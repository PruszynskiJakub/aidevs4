Key Design Properties

Crash-safe
All state persisted to SQLite before waiting. Process can restart — reconcileOrphanedWaits() at startup finds and resumes stranded runs.
Idempotent resume
Optimistic locking (version field) prevents double-resume. Safe to call resumeRun() multiple times.
Granular decisions
Each tool call in the batch gets its own approve/deny decision. Partial approval is supported.
Multi-surface
Wait/resume protocol is transport-agnostic. Works from CLI (inline provider), HTTP API, or Slack bot.

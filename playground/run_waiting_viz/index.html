Takeaway. On the left, the parent turn loop has only two options when it hits delegate_to_agent: block synchronously (nested loop, loses cancellation, dies on process restart) or fire-and-forget (parent never sees the result). On the right, the parent transitions to waiting with a typed child_run descriptor, persists, and returns from the loop. A resume handler — triggered when the child completes — injects the result as a tool message and flips the parent back to running. The same machinery handles HITL confirmations (user_approval) and sandbox writeback review (sandbox_writeback_approval). One primitive, three unblocked capabilities.

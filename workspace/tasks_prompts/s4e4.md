## Mission

Organize Natan's notes into a structured virtual filesystem via the hub API. The filesystem must describe trade relationships: which cities need which goods, who manages trade in each city, and which goods are offered for sale by which cities.

## Data

Download Natan's notes from `https://hub.ag3nts.org/dane/natan_notes.zip`. Unzip and read every file to understand the trade network described in them.

You can preview the current filesystem state at: `https://hub.ag3nts.org/filesystem_preview.html`

## API

All filesystem operations go through hub verify with task name `filesystem`.

Start by calling the help action to discover available commands:

```json
{"action": "help"}
```

Submit this as the answer via hub verify (task: "filesystem"). The API will return available actions (createFile, createDir, list, delete, reset, done, etc.).

### Batch mode

You can send multiple operations in one request by making the answer a JSON **array** of action objects:

```json
[
  {"action": "createDir", "path": "/cities"},
  {"action": "createFile", "path": "/cities/krakow", "content": "..."}
]
```

## Required Structure

Build three directories with these exact names: `/miasta`, `/osoby`, `/towary`

### /miasta (cities)

One file per city mentioned in the notes. Filename = city name in nominative case, no Polish diacritics.

File content = JSON object mapping goods to quantities (no units). Example:
```json
{"zboze": 50, "drewno": 120}
```

These represent what each city **needs** (demand side).

### /osoby (people)

One file per person responsible for trade in a city. Filename = person's name with underscores instead of spaces, no Polish diacritics.

File content = the person's full name and a markdown link to their city file. Example:
```
Jan Kowalski [krakow](/miasta/krakow)
```

### /towary (goods)

One file per trade good that is **offered for sale**. Filename = good name in nominative singular, no Polish diacritics (e.g. "koparka" not "koparki").

File content = markdown link to the city that sells this good. Example:
```
[krakow](/miasta/krakow)
```

## Important Rules

- No Polish diacritics in filenames or JSON keys (a not a with ogonek, c not c with acute, etc.)
- Good names in nominative singular form
- City names in nominative case
- JSON values should also avoid Polish diacritics

## Strategy

1. Call `help` first to learn the exact API actions and their parameters.
2. Download and extract the notes ZIP. Read every note file carefully.
3. Use the think tool to build a complete picture of the trade network:
   - List all cities and what goods each one needs (with quantities)
   - List all people and which city each one manages
   - List all goods for sale and which city sells each one
   - Pay close attention to the difference between what a city **needs** (demand, goes in /miasta) vs what a city **sells/offers** (supply, goes in /towary)
4. Reset the filesystem to start clean.
5. Create all directories and files in one batch request.
6. Call `done` to submit for verification.

If verification fails, list the filesystem contents, compare against the notes, fix issues, and retry.

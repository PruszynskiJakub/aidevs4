### What's queued — pick this up first
**`_cases/production_hardening_audit.md`** is the next chunk of work.
It's an audit of `src/` against `Wonderlands/apps/server`, framed as:
"when this layer fails / crashes mid-flight / runs concurrently with

across 10 dimensions) are already written. P0s spotted today that are
still open:

- **Idempotency (1.1, 1.2)**: HTTP routes have no idempotency-key
  handling; tool calls to side-effecting external APIs have none either.
- **Error taxonomy (2.2)**: 25+ silent `catch {}` blocks across `src/`

  only raw throws.

  not gated by run lifecycle. SP-94 covered DB; file/atomic and memory
  gating are still open.
- **Concurrency (4.1)**: HTTP `/chat` uses a per-session promise queue
  but every other entry point (Slack, CLI, scheduled jobs) bypasses it.
- **Resilience (5.3, 5.4)**: no request body size cap on Hono routes;
  no request rate limiting. Important since this is going on a public
  VPS.
- **Input validation at boundaries (6.1–6.4)**: HTTP/Slack/MCP/Hub
  responses aren't schema-validated.
- **Secrets / PII (7.1, 7.2, 7.4)**: Langfuse subscriber sends every
  tool input and LLM message verbatim; error messages leak filesystem
  paths; stack traces expose internal paths.
- **Process lifecycle (9)** and **crash recovery (10)** dimensions are
